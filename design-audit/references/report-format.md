# Report Format

## JSON structure

Emit this JSON first (used for programmatic consumption and Linear body generation):

```json
{
  "generated_at": "2026-04-20T09:00:00Z",
  "commit_sha": "<read from git rev-parse HEAD>",
  "summary": {
    "errors": 0,
    "warnings": 0,
    "info": 0,
    "coverage_pct": 0
  },
  "missing_docs": [
    { "ui_file": "src/ui/banner.tsx", "component": "banner" }
  ],
  "violations": [
    {
      "file": "src/app/(dashboard)/settings/page.tsx",
      "line": 42,
      "category": "substitution",
      "severity": "warn",
      "rule_id": "use-button-primitive",
      "suggestion": "Use <Button> from @/ui/button",
      "design_ref": "/design/components/button"
    }
  ],
  "pattern_candidates": [
    {
      "proposed_name": "empty-state",
      "occurrences": ["src/app/(dashboard)/emails/page.tsx:88", "src/app/(dashboard)/domains/page.tsx:34"],
      "rationale": "Icon + heading + description + CTA repeated across 3+ dashboard pages without a documented pattern."
    }
  ]
}
```

**`coverage_pct`** = `(documented_count / total_ui_components) * 100`, rounded to one decimal.
**`summary.errors/warnings/info`** = counts of `violations` (not `missing_docs`) by severity. Missing docs always count as warnings toward the summary total.

---

## Markdown report template

Render this after the JSON. This is what goes into the Linear issue body.

```markdown
## Design Audit — {DATE}

**Commit:** `{SHA_SHORT}` | **Coverage:** {COVERAGE_PCT}% ({DOCUMENTED}/{TOTAL} components documented)

**Summary:** {ERRORS} errors · {WARNINGS} warnings · {INFO} info

---

### Missing documentation ({COUNT})

{IF NONE}
_All `src/ui/` components are documented._ ✅
{ELSE}
| Component | File |
|-----------|------|
| `{component}` | `{ui_file}` |
...

> Add documentation pages at `src/app/(internal)/design/components/<name>/page.tsx`.
{END IF}

---

### Violations ({COUNT})

{IF NONE}
_No violations found._ ✅
{ELSE}
<details>
<summary>{ERRORS} errors, {WARNINGS} warnings, {INFO} info</summary>

| File | Line | Rule | Severity | Suggestion |
|------|------|------|----------|------------|
| `{file}` | {line} | `{rule_id}` | {severity} | {suggestion} — [{design_ref}]({design_ref}) |
...

</details>
{END IF}

---

### Pattern candidates ({COUNT})

{IF NONE}
_No recurring compositions detected that lack a documented pattern._
{ELSE}
These compositions appear in 3+ files and could be extracted as documented patterns:

<details>
<summary>Candidates</summary>

{FOR EACH CANDIDATE}
**`{proposed_name}`**
{rationale}
Occurrences: {occurrences joined by ", "}

{END FOR}
</details>
{END IF}

---

_Generated by the `design-audit` skill · [Design system](/design) · [Audit rubric](design-audit/references/rubric.md)_
```

Fill in all `{PLACEHOLDER}` values from the JSON report before filing the issue.
