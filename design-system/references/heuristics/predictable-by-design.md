# Predictable by design

> **Status: TODO.** Notion source has bullets but no finalised guidance. Treat the points below as drafts and escalate to `@design` for anything binding.

## Draft notes from Notion

- Avoid confusion by using consistent patterns. Users learn once and then expect the same shape everywhere.
- API ↔ dashboard parity matters: divergence is jarring.
  - Example of "not ideal": broadcast HTML is editable via the API but not in the dashboard.

## Open questions

- Where parity is required vs where it's acceptable for the API and dashboard to differ (compare with [`api-first`](api-first.md)).
- A checklist for "is this consistent with the rest of the dashboard?" reviewers can apply.
