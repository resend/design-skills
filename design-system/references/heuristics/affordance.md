# Affordance

> **Status: TODO.** Notion source has bullets but no finalised guidance. Treat the points below as drafts and escalate to `@design` for anything binding.

## Working definition

> Affordances are the characteristics or properties of an object that suggest how it can be used. They show a user that an object can be interacted with.
>
> An affordance is not a "property" of an object — it lives in the relation between the user and the object. A door affords opening if you can reach the handle. For a toddler who can't reach the handle, the same door doesn't afford opening.

## Draft notes from Notion

- A button labelled "Export" that's active for non-admins is a broken affordance — it suggests it can be used when it can't.
- A disabled button should not contain an animated icon — animation suggests interactivity the user doesn't have.
- External links and internal links should look distinguishable so the user can predict where a click will take them.
